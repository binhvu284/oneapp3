# Contributing to OneApp

First off, thank you for considering contributing to OneApp!

## Development Workflow

1. **Branch Naming**: 
   - `feat/feature-name`
   - `fix/bug-fix`
   - `docs/doc-updates`
2. **Commit Guidelines**:
   - Write clear and meaningful commit messages.
   - The project uses `husky` and `lint-staged`. Your code will be automatically linted and formatted on commit.
3. **Tests**:
   - Make sure your changes are tested (`npm run test`).
4. **Pull Requests**:
   - Submit PRs against the `main` branch. Provide a detailed summary of your changes.

Happy Coding!
